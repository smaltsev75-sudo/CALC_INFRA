## Термины и сокращения

В калькуляторе используются стандартные финансовые и доменные сокращения. Если встретите незнакомое — загляните в таблицу ниже.

### Финансовые

| Сокращение | Расшифровка | Что значит |
|---|---|---|
| **CAPEX** | Capital Expenditures | Капитальные затраты — разовые / долгосрочные вложения: закупка лицензий, внедрение, пентесты, сертификация. Списываются на проект целиком, не помесячно. |
| **OPEX** | Operating Expenses | Операционные затраты — регулярные ежемесячные расходы: аренда облачной инфраструктуры, оплата токенов ИИ, поддержка, обновления, SLA-обслуживание. |
| **НДС** | Налог на добавленную стоимость | Российский налог: 18 % до 2019 года, 20 % с 2019 по 2025, 22 % с 01.01.2026. В калькуляторе — отдельная независимая ось расчёта. |

### Технологические

| Сокращение | Расшифровка | Что значит |
|---|---|---|
| **ИИ** (AI) | Artificial Intelligence — искусственный интеллект | В калькуляторе — затраты на токены большой языковой модели, эмбеддинги, поисковую базу, виртуальные ядра и память для AI-агентов. |
| **LLM** | Large Language Model — большая языковая модель | Например, GigaChat, GPT. Считается стоимость её использования по количеству входных и выходных токенов. |
| **RAG** | Retrieval-Augmented Generation — поиск с дополнением | Поисковая база при большой языковой модели: даёт ей контекст из вашей документации. Считается размер корпуса и число эмбеддингов. |
| **SLA** | Service Level Agreement — соглашение об уровне обслуживания | Целевой процент доступности сервиса (99.5 % / 99.9 % / 99.95 %). Чем выше — тем дороже инфраструктура (резервирование, мониторинг). |

### Метрики нагрузки

| Сокращение | Расшифровка | Что значит |
|---|---|---|
| **DAU** | Daily Active Users — суточная аудитория | Количество уникальных пользователей, выполнивших хотя бы одно целевое действие за календарные сутки. Считается без повторов в пределах суток. Косвенно характеризует частоту использования продукта. |
| **RPS** | Requests Per Second — пиковая частота запросов | Среднее количество HTTP-запросов (или эквивалентных вызовов) к серверу в секунду в пиковый момент рабочих суток. Определяет требования к вычислительным ресурсам: количеству виртуальных ядер, объёму оперативной памяти, пропускной способности сети. Считается как агрегат по всем активным сессиям в пик. |

### Регуляторика РФ

| Сокращение | Расшифровка | Что значит |
|---|---|---|
| **152-ФЗ** | Федеральный закон № 152-ФЗ «О персональных данных» | Российский закон, регулирующий обработку персональных данных. От уровня категории ПДн (К1–К4) зависят требования к безопасности и стоимость средств защиты. |
| **ФСТЭК** | Федеральная служба по техническому и экспортному контролю | Российский регулятор в области информационной безопасности. Сертифицирует средства защиты и устанавливает требования к классам защищённости. |

## Структура интерфейса

Слева — боковое меню (сворачивается до 64 пикселей на узких экранах). Сверху — верхняя панель с именем активного расчёта и быстрыми кнопками `Импорт JSON` / `Экспорт JSON` / `PDF` / `Сброс` / переключателем темы.

| Раздел | Назначение |
|---|---|
| **Расчёты** | Список расчётов: создать / открыть / дублировать / удалить / экспорт-импорт. На карточке — имя, дата, общая сумма, режим («С рисками» / «Без рисков»). |
| **Опросник** | ~80 вопросов о продукте + блок «Параметры расчёта» (буферы, риски, НДС, размеры стендов, AI-нагрузка). |
| **Дашборд** | Главный блок «Итого по расчёту», «Сводка состояния расчёта» (готовность / качество / бюджет / следующий шаг / оптимизация стоимости — один блок), распределение по категориям, вклад риск-коэффициентов, 5 стенд-карточек. |
| **Детализация** | Подвкладки «Бюджет (₽)» / «Объём (qty)», построчно каждый ЭК × стенд + кнопка CSV. |
| **Сравнение расчётов** | 2–4 расчёта рядом с дельтой по стендам и категориям. |

При включённом переключателе **«Расширенные настройки»** (в нижней части бокового меню, рядом со «Справкой») появляется ещё одна группа — **Администрирование** с разделами «Элементы» (управление каталогом) и «Вопросы» (управление опросником). По умолчанию скрыто. Состояние переключателя переживает F5.

## Типовой сценарий использования

```
1. Расчёты → Новый расчёт (имя проекта)
        ↓
2. Мастер быстрого старта (3 пресета) → быстрый старт с типовыми ответами
        ↓
3. Опросник → донастройка ответов (5–10 минут)
   • Пользователи / активные в день / запросы в секунду
   • База данных / файловое хранилище
   • Email / SMS / PUSH-уведомления
   • Безопасность (152-ФЗ, ФСТЭК)
   • Пентесты, нагрузочное тестирование
   • ИИ / большая языковая модель / поисковая база (если нужны)
        ↓
4. Опросник → блок «Параметры расчёта»
   • Длительность фазы
   • Буферы (задачный + проектный)
   • Риск-коэффициенты
   • НДС / горизонт планирования
   • Размеры стендов (% от ПРОМ)
        ↓
5. Дашборд + Детализация — посмотреть результат
        ↓
6. Анализ расчёта — Качество / Допущения / Чувствительность / Бюджет
        ↓
7. Следующие шаги (блок на Дашборде) — что разумно сделать дальше
        ↓
8. Обоснование расчёта / PDF / JSON / CSV — поделиться результатом с командой
        ↓
9. Сравнение расчётов — несколько вариантов рядом
```

**Совет.** Если первый расчёт даёт «слишком дорого / слишком дёшево» — откройте **Детализацию**, поищите аномальные строки. Часто это:
- не указано «Пиковая нагрузка в запросах в секунду» в Опроснике → количество виртуальных машин считается от 0;
- лицензия СУБД на много виртуальных ядер при низком сценарии;
- разовые работы (внедрение / пентест) разнесены равномерно по длительности фазы — увеличение фазы их «размазывает».

## Расчёты

### Создание расчёта

**Расчёты → Новый расчёт.** Имя — это имя проекта. Открывается мастер быстрого старта с тремя пресетами:

| Пресет | Когда выбрать |
|---|---|
| **Стандартный** | Типовое корпоративное облачное приложение (B2B SaaS-приложение) с умеренной нагрузкой. |
| **Высокая нагрузка** | Высоконагруженные сервисы (электронная коммерция, финтех) с пиковыми сценариями. |
| **Внутренний инструмент** | Корпоративное приложение для сотен пользователей. |

После выбора пресета — диалог уточнения (152-ФЗ / тип приложения / аудитория) и расчёт сразу переходит в Опросник, заполненный на ~70%. Дальше вы донастраиваете ответы под свой продукт.

### Сценарии внутри расчёта

Один расчёт может содержать несколько **сценариев** — альтернативные ветви ответов внутри той же модели. Сценарии нужны, когда вы хотите оценить **тот же продукт в нескольких конфигурациях**, не плодя десяток дублирующихся расчётов и сохраняя общие настройки (провайдер, НДС, риски) синхронно между ними.

**Чем расчёт отличается от сценария.**

| | Расчёт | Сценарий |
|---|---|---|
| Уровень | Отдельный проект верхнего уровня (карточка в разделе «Расчёты») | Ветвь внутри расчёта |
| Файл JSON-экспорта | Один файл = один расчёт со всеми его сценариями | Самостоятельного файла нет |
| Имя в верхней панели | Да (заголовок) | Нет (видно во вкладках ниже) |
| Когда нужен | Разные проекты / клиенты / продукты | Один продукт, разные конфигурации |

#### Как выглядит в интерфейсе

Между названием расчёта и кнопками `Импорт JSON / Экспорт JSON / PDF` живёт горизонтальная полоса вкладок:

```
[Базовый •] [Минимальная версия] [Пилот 100 пользователей ⋯]  [+ Сценарий]
```

- Активная вкладка подсвечена.
- Маркер «**•**» рядом с названием — в этом сценарии есть **ручные правки** (поля со значком «Вы изменили»). Всплывающая подсказка на точке показывает количество.
- Кнопка `⋯` справа от названия — действия со сценарием: переименовать / дублировать / удалить.
- `+ Сценарий` справа — добавить новый.

Полоса вкладок видна **на всех экранах** (Опросник / Дашборд / Детализация / Сравнение). Переключение мгновенно меняет содержимое всего приложения — Дашборд и Детализация пересчитываются под ответы активного сценария.

#### Что общее, что персональное

Это важно понимать заранее, чтобы не удивляться, когда правка в одном сценарии «перетекла» в другой.

| Что | Где живёт | Поведение |
|---|---|---|
| Ответы опросника (пользователи, нагрузка, ИИ, безопасность и т.д.) | В каждом сценарии — своё | Правка в одном сценарии **не** меняет другие |
| Метки происхождения ответов («Из профиля» / «Из масштаба» / «Вы изменили») | В каждом сценарии — своё | Не переносятся между сценариями |
| Профиль мастера быстрого старта (если был выбран при создании) | В каждом сценарии — свой | Каждый сценарий помнит свой профиль независимо |
| Глобальные настройки: НДС, провайдер, активный прайс, риск-коэффициенты, инфляция, сезонность, резерв на непредвиденное, горизонт планирования | Одно на весь расчёт | Меняется во всех сценариях одновременно |
| Размеры стендов, распределение нагрузки по ресурсам, AI-нагрузка на стендах | Одно на весь расчёт | Меняется во всех сценариях одновременно |
| Настройки отображения (отключённые стенды, активная вкладка, развёрнутые секции опросника) | Одно на весь расчёт | Меняется во всех сценариях одновременно |
| Каталог ЭК и список вопросов (если редактировали в Администрировании) | Одно на весь расчёт | Меняется во всех сценариях одновременно |

**Практический вывод.** Если вы хотите сравнить **«с НДС и без НДС»** или **«Cloud.ru и Yandex Cloud»** — это **разные расчёты**, не сценарии (переключатель НДС / провайдер — общие). Если же сравниваете **«Минимальная версия 100 пользователей и Промышленная версия 10 000 пользователей»** при том же провайдере и тех же рисках — это **сценарии**.

#### Действия со сценариями: создание, переключение, переименование, удаление

| Действие | Как сделать |
|---|---|
| **Создать сценарий** | Кнопка `+ Сценарий` в полосе вкладок. Открывается модалка переименования с пустым полем; впишите имя и нажмите Enter. **Новый сценарий наследует профиль быстрого старта от активного**, но стартует с **пустыми ответами**. Семантика: «тот же продукт, другая конфигурация». Если нужна копия с ответами — используйте «Дублировать». |
| **Переключить активный** | Клик по вкладке. Интерфейс мгновенно пересчитывается. Всплывающее уведомление: «Сценарий переключён». |
| **Переименовать** | `⋯` → «Переименовать». Имя обрезается до 60 символов. |
| **Дублировать** | `⋯` → «Дублировать». Открывается модалка с именем по умолчанию `«<источник> (копия)»` — можно принять или ввести своё. Копируются и профиль быстрого старта, и ответы, и метки происхождения. |
| **Удалить** | `⋯` → «Удалить». Модалка подтверждения с явным предупреждением. **Последний сценарий удалить нельзя** — кнопка появится только при ≥ 2 сценариях. Глобальные настройки расчёта при удалении сценария **не теряются**. |

Все операции сохраняются в браузере сразу же; индикатор сохранения в верхней панели мигает «Сохранено».

#### Примеры использования

**Пример 1 — Поэтапный запуск продукта.**
Команда планирует запустить продукт в три этапа и хочет видеть полную стоимость каждой фазы при тех же риск-параметрах.

```
Расчёт «Мой продукт»
  ├─ Сценарий «Пилот 50 пользователей»
  │     активных в день 30, запросов в секунду 5, БД 20 ГБ, без ИИ
  ├─ Сценарий «Минимальная версия 1 000 пользователей»
  │     активных в день 400, запросов в секунду 80, БД 150 ГБ, поисковая база включена
  └─ Сценарий «Промышленная версия 10 000 пользователей»
        активных в день 4 000, запросов в секунду 600, БД 2 ТБ, поисковая база + дообучение модели
```

НДС, провайдер, инфляция, риск-коэффициенты — одинаковые во всех трёх. Переключаетесь между вкладками — видите стоимость каждой фазы. Сценарий «Сравнить сценарии» (см. ниже) положит их в одну таблицу.

**Пример 2 — Чувствительность к AI-нагрузке.**
Базовый сценарий без ИИ + два сценария с разной интенсивностью ИИ для оценки «сколько стоит AI-функция».

```
  ├─ Базовый — большая языковая модель выключена
  ├─ ИИ лёгкая — 10 % пользователей пользуются ИИ, 3 запроса в день на каждого
  └─ ИИ тяжёлая — 80 % пользователей пользуются ИИ, 30 запросов в день на каждого
```

Сравнение между сценариями сразу показывает дельту по категориям «ИИ» и «Лицензии», не трогая прочие категории.

**Пример 3 — Оптимизация одного риск-параметра.**
Базовый сценарий с реалистичным риск-набором + дубль с агрессивным сценарием понижения буферов.

```
  ├─ Реалистичный — задачный буфер 30 %, проектный буфер 20 %, резерв на непредвиденное 15 %
  └─ Агрессивный  — задачный буфер 10 %, проектный буфер 10 %, резерв на непредвиденное 5 %
```

⚠ **Важная оговорка для этого кейса.** Риск-коэффициенты — **глобальные настройки** (см. таблицу выше), они НЕ перенесутся между сценариями. Чтобы сравнить риск-настройки, удобнее использовать **отдельные расчёты** или **План оптимизации стоимости** (карточка на Дашборде), который явно работает с настройками рисков в пределах одного расчёта.

Для изменений внутри сценария подходят: количество пользователей, параметры ИИ, политики безопасности, срок хранения данных, частота резервного копирования — всё, что живёт в ответах опросника.

**Пример 4 — Заявка и реальный масштаб.**
Заказчик сказал «1 000 пользователей», но архитектор подозревает, что через год будет 10 000. Делаем оба варианта рядом:

```
  ├─ Сценарий «Заявленный — 1 000 пользователей» (на базе пресета «Стандартный»)
  └─ Сценарий «Реалистичный — 10 000 + рост 50 %/год»
```

Получаем две оценки совокупной стоимости рядом и используем для аргументации с бизнесом. Дубликат с правкой пары полей делается за минуту.

#### Сравнение сценариев

При двух и более сценариях на Дашборде в блоке **«Следующие шаги»** появляется карточка **«Сравнить сценарии»** → открывает модалку «Сравнение сценариев»: каждая ваша конфигурация показана отдельной колонкой, по строкам — стоимость по стендам и категориям. Эта модалка также позволяет одновременно выбрать несколько провайдеров и увидеть перекрёстную таблицу «сценарии × провайдеры».

Альтернатива без модалки — просто переключайте вкладки сценариев и сравнивайте Дашборд / Детализацию глазами. Часто этого достаточно.

#### Подводные камни и частые вопросы

- **Удалил активный сценарий — куда переключится?** На первый из оставшихся в списке. Последний сценарий удалить нельзя.
- **Изменил настройку (НДС / провайдер) — она поменялась в других сценариях.** Так задумано: это общие настройки. Если хотите изолировать настройки — это **разные расчёты**, не сценарии.
- **При экспорте JSON попадает только активный сценарий?** Нет, в JSON-файле сохраняется **весь расчёт со всеми сценариями** + указатель на активный. При импорте всё восстановится.
- **Где видны «ручные правки» сценария?** Точка-индикатор на вкладке (только при ≥ 1 ручной правке). Сами поля помечены значком «Вы изменили» в Опроснике рядом с полем ввода.
- **Повторное применение профиля быстрого старта затронет другие сценарии?** Нет, **только активный сценарий**. Каждый сценарий помнит свой профиль и свои ручные правки независимо.
- **Можно ли переименовать сценарий на пустую строку?** Нет — действие просто не сработает, имя останется прежним. Длина имени ограничена 60 символами.
- **Что делать, если случайно удалил сценарий?** Только если есть резервная копия JSON-файла — повторно импортировать. Локальной истории удалённых сценариев нет.
- **На Дашборде вместо кнопки «Изменить параметры» вижу «Задать профиль сценария» — почему?** У этого сценария не задан профиль быстрого старта. Это бывает для сценариев, созданных до версии 2.13.1 (раньше кнопка `+ Сценарий` создавала пустой профиль), либо при импорте JSON без профиля. Нажмите кнопку — откроется мастер быстрого старта с предзаполненными значениями по умолчанию, после подтверждения профиль и связанные ответы применятся к этому сценарию (другие сценарии не затрагиваются). С версии 2.13.1 новые сценарии через `+ Сценарий` сразу наследуют профиль активного, поэтому это пустое состояние встречается только для старых расчётов.

### Что считается

```
costFinal = qty × pricePerUnit × billingIntervalMul × (риск-факторы) × НДС
```

где:
- `qty` — количество единиц для каждой пары (ЭК × стенд), вычисляется по формуле из ответов опросника.
- `pricePerUnit` — цена за единицу из каталога.
- `billingIntervalMul` — нормализация к месяцам (`daily / monthly / annual / oneTime`).
- Риск-факторы — буферы, инфляция, сезонность, сдвиг сроков, непредвиденные.
- НДС — независимая от рисков ось (включается отдельным переключателем).

Полная схема и список риск-коэффициентов — в [Architecture.md](Architecture.md).

### Режим «Без рисков»

Переключатель **«Учитывать риск-коэффициенты в бюджете»** в **Опросник → Параметры расчёта**. При выключении 7 связанных полей (буферы, инфляция, сезонность, расписание, непредвиденные) приглушаются. НДС применяется независимо.

В режиме «Без рисков»:
- На дашборде метка в главном блоке меняется `С РИСКАМИ` → `БЕЗ РИСКОВ` (зелёный → оранжевый).
- В карточке «Вклад риск-коэффициентов» показывается **потенциальная** наценка.
- В Детализации заголовок «Стоимость, ₽» получает метку «Без рисков».

### Режимы НДС

Ставка НДС в РФ — параметр с историей (18% до 2019, 20% с 2019 по 2025, 22% с 01.01.2026). Калькулятор поддерживает три режима её определения (поле в **Опросник → Параметры расчёта → НДС**):

- **Авто (по дате)** — ставка берётся из встроенного справочника по дате создания расчёта. Дефолт для новых расчётов. Значок рядом с полем: «Авто 22% · дд.мм.гггг».
- **Вручную** — пользователь задаёт особую ставку (нерезидент, экспорт, льгота). Не пересчитывается автоматически. Значок: «Вручную 18%».
- **Заморожено** — ставка зафиксирована для согласованного бюджета. Обновления приложения не должны менять итог. Значок: «Заморожено 20%, дд.мм.гггг».

Переключение режимов — кнопки «Авто» / «Вручную» / «Заморозить» рядом со значком. Прямая правка поля «Ставка НДС, %» автоматически переводит расчёт в режим «Вручную».

**Предупреждение при пересечении даты изменения ставки.** Если ваш горизонт планирования пересекает дату, когда ставка НДС меняется (например 1 января 2026 года: 20% → 22%), приложение покажет уведомление (не блокирующее) под полем НДС:

> Расчёт пересекает дату изменения НДС: 01.01.2026, 20% → 22%. Сейчас применяется ставка на дату, указанную для расчёта НДС; для даты в 2025 году это 20%. Для точной оценки разделите бюджет по периодам или задайте ставку вручную.

В текущей версии применяется одна ставка (на дату ставки расчёта); смешанный режим (взвешенное усреднение по месяцам горизонта) не реализован.

**Расчёты прежних версий (созданные до 1 января 2026 года, обычно при ставке 20%).** При первом открытии в новой сессии вы увидите информационное сообщение:

> Расчёт создан при ставке НДС 20%. Ставка зафиксирована, чтобы не изменить согласованные цифры. Сменить режим можно в Опроснике.

Это сделано специально: суммы старых согласованных бюджетов не меняются автоматически при обновлении приложения. Если нужно — переключите режим вручную в Опроснике.

**Где режим НДС виден:**
- В Опроснике — значок + 3 кнопки.
- В обосновании расчёта (раздел «Краткое резюме») — строка вида «Ставка НДС: 22% (авто, дата ставки: дд.мм.гггг)».
- В Сравнении — мелкая метка «НДС 22% · авто» под именем каждого расчёта; предупреждение «Ставки НДС различаются — итоги не сопоставимы напрямую» при сравнении расчётов с разными ставками.

### Стенды

| ID | Название | Дефолтный размер (% от ПРОМ) | Допустимый диапазон |
|---|---|---|---|
| DEV | DEV | 20% | 5–100% |
| IFT | ИФТ | 40% | 5–100% |
| PSI | ПСИ | 50% | 5–100% |
| LOAD | НТ | **120%** | 20–120% |
| PROD | ПРОМ | **100% (зафиксировано)** | — |

ПРОМ — эталон, к которому привязаны остальные стенды. Стенды DEV/ИФТ/ПСИ ограничены 100% — они представляют собой функциональные копии промышленной среды и не имеют смысла превышать её. Стенд **НТ** может превышать ПРОМ (до 120%) — нагрузочное тестирование в стресс-режиме требует мощности выше предполагаемой промышленной нагрузки, чтобы найти пределы системы. AI-нагрузка на стендах (отдельная настройка) ограничена 0–100% для всех стендов — это доля от промышленного эквивалента, не запас мощности.

### Распределение нагрузки по ресурсам

В **Опросник → Параметры расчёта → Размеры стендов** есть таблица 4×6 (DEV/ИФТ/ПСИ/НТ × CPU/GPU/RAM/SSD/HDD/S3). Это позволяет, например, для DEV установить CPU=20%, RAM=10%, SSD=5% — стенды получат разную долю каждого ресурса вместо единого процента.

Элементы конфигурации без указанного типа ресурса (Услуги, Лицензии, Безопасность, Трафик) используют общий размер стенда.

### AI-нагрузка на стендах

Главный переключатель «AI-агенты включены» + индивидуальный коэффициент для каждого стенда:

| Стенд | Дефолт |
|---|---|
| DEV | 2% |
| ИФТ | 20% |
| ПСИ | 50% |
| НТ | 100% |
| ПРОМ | 100% (заперт) |

DEV ≠ 0 — это AI-трафик разработки (регрессионные тесты, разовые проверки, демо). Применяется ко всем AI-метрикам одинаково: токены, эмбеддинги, виртуальные ядра для агентов, память агентов.

## Анализ расчёта

Анализ расчёта — это группа инструментов, которые помогают понять, насколько ваш расчёт «хороший» и где тонкие места. Все они доступны с Дашборда.

### Качество расчёта (проверка)

**Дашборд → карточка «Качество расчёта» → Открыть проверку расчёта →**. Карточка показывает только состояние оценки и две прямые кнопки: «Открыть проверку расчёта» и «Уточнить данные» (мастер дозаполнения). Переходы к допущениям, чувствительности и обоснованию вынесены в блок «Следующие шаги», чтобы из карточки шёл один путь к одному действию.

Показывает оценку 0–100 на основе:
- доли заполненных вопросов;
- количества рискованных допущений «по умолчанию»;
- количества ошибок в ответах (несовместимые сочетания, отрицательные значения).

Внутри модалки — список конкретных предупреждений и ошибок с переходом на источник (на конкретное поле в Опроснике).

### Динамика качества

В той же модалке — секция **«Динамика качества»**: лента последних 20 точек, дельта «было → стало», лучшее значение. Снимок записывается:
- при открытии проверки качества;
- при завершении шага в Мастере уточнения;
- при открытии действия из «Следующих шагов».

Хранится локально (в браузере), при экспорте JSON-файла **не передаётся** — это ваша приватная история действий.

### Реестр допущений

**Дашборд → значок «Допущения (N)» в главном блоке**.

Список ответов, на которые вы не дали явного значения и калькулятор использовал значение по умолчанию для случая «Не знаю». Помогает увидеть «сколько у меня сейчас догадок и какие из них рискованные» (рискованные подсвечены).

### План оптимизации стоимости

**Дашборд → «Сводка состояния расчёта» → секция «Оптимизация стоимости» → `Открыть план оптимизации`**.

Инструмент для владельца продукта и Архитектора: показывает, **за счёт чего можно снизить стоимость и какой ценой**, и позволяет применить выбранные правки к расчёту одним явным действием.

Начиная с версии 2.16, отдельная карточка «План оптимизации стоимости» на Дашборде удалена — точка входа встроена внутри «Сводки состояния расчёта». Вся работа происходит в модалке. Если бюджет превышен и кнопка «Следующий шаг» уже ведёт в планировщик оптимизации, дублирующая кнопка-тизер скрывается — действие остаётся одно («одна цель — одно действие»).

#### Что важно знать сразу

- **Это не «автооптимизатор» и не «применить план целиком».** Вы редактируете конкретные параметры (значения слайдеров, дней хранения, % буфера, целевой уровень SLA), и только после нажатия `Применить изменения` они уходят в расчёт.
- **До явного применения активный расчёт не меняется.** Открытие модалки, переключение уровня, изменение ограничений, правка значений — всё это живёт в **черновике**. Дашборд / Детализация / PDF продолжают показывать прежний расчёт.
- **Черновик переживает закрытие модалки**, но **теряется при F5** (живёт только в текущей сессии, без сохранения в браузере). Это сделано осознанно: оптимизация — это «сессия принятия решения», не постоянная сущность.
- **Откат тоже работает только в текущей сессии.** После применения в модалке появляется ярлык «Откатить последнее применение» — он восстанавливает состояние, какое было до применения, пока модалка открыта. F5 либо новое применение сотрут снимок.

#### Структура модалки

1. **Уровень оптимизации** — три вкладки сверху:
   - **Консервативный** (0–5 %) — минимальная боль, без снижения целевого уровня SLA, без снижения ИИ и поисковой базы.
   - **Амбициозный** (5–15 %) — управляемый компромисс, ИИ и поисковую базу можно уменьшать, снижение целевого уровня SLA остаётся опциональным.
   - **Экстремальный** (15–25 %) — пересмотр требований, ИИ и поисковую базу можно уменьшать, снижение целевого уровня SLA остаётся опциональным.

   По умолчанию — **Амбициозный**. Уровень задаёт три вещи: целевой диапазон экономии, агрессивность подсказок «Рекомендуется» и **стартовый набор ограничений**.

2. **Ограничения** — шесть переключателей («Можно снижать SLA», «Можно уменьшать стенды», «Можно снижать риск-буферы», «Можно уменьшать ИИ / поисковую базу», «Можно уменьшать срок хранения данных», «Защитить требования соответствия»). Переключатель фильтрует доступные рычаги.

   **Поведение при смене уровня:** при первом выборе уровня применяются его ограничения по умолчанию. Как только вы **явно** нажали переключатель, дальнейшая смена уровня **не перетрёт** ваш выбор — ваше «можно снижать SLA» останется, даже если переключиться на Консервативный. Не тронутые ограничения по-прежнему берутся от уровня. Маркер «·» рядом с подписью показывает «вы явно тронули этот переключатель».

3. **Итог** — три карточки (текущая стоимость / после изменений / экономия) + строка статуса + **переключатель периода** (день / месяц / год):
   - «Изменений пока нет.»
   - «Попадает в диапазон 5–15 %.»
   - «Пока 3.8 % — ниже цели 5–15 %. Добавьте рычаги.»
   - «11.1 % выше верхней границы 15 %. Возможно, выбранные изменения ближе к следующему уровню.»
   - «Не удалось пересчитать черновик. Проверьте значения.»

   Все числа пересчитываются на полной копии расчёта (не оценочно). Базовая стоимость берётся из текущего активного расчёта. **Суммы всегда показываются в тыс. ₽** для согласованности с дашбордом и снижения визуального шума. Переключатель периода при первом открытии модалки берёт текущий период дашборда; дальше живёт независимо.

4. **Рычаги** — сгруппированы по **областям компромисса** (что вам придётся уступить):
   - **Инфраструктура и стенды** — разрешение «Можно уменьшать стенды» открывает рычаги для DEV / ИФТ / ПСИ / НТ-стенда.
   - **Надёжность и SLA** — разрешение «Можно снижать SLA».
   - **Данные и хранение** — разрешение «Можно уменьшать срок хранения» для резервных копий и журналов.
   - **ИИ / поисковая база** — разрешение «Можно уменьшать ИИ / поисковую базу» для исходящих токенов, корпуса, эмбеддингов.
   - **Риски и резервы** — разрешение «Можно снижать риск-буферы» для задачного и проектного буферов, резерва на непредвиденное, запаса на сдвиг расписания.
   - **Горизонт планирования** — без ограничения, рычаг доступен только если горизонт > 3 лет и не на уровне «Консервативный».

   Каждая группа — **сворачиваемый блок** с заголовком, показывающим сводку: количество доступных параметров, накопленную экономию по группе (`−X тыс. ₽ / мес`), максимальный риск внутри группы. Экономия по группе считается **точно** — отдельный пересчёт на полной копии расчёта, где применены только изменения этой группы (а не приблизительная эвристика).

   **Заблокированные группы** (соответствующее разрешение выключено) **не исчезают** — заголовок показывает значок замка, тело видно с причиной «Заблокировано: ...» и встроенной кнопкой `Разрешить ...`. Кнопка только снимает ограничение (не применяет правки) — это короткий путь без прокрутки к блоку «Ограничения» наверху.

   **Пустые группы** (разрешение включено, но условия применимости отсеяли все рычаги — например, группа «ИИ» когда большая языковая модель не используется) сортируются в самый низ списка и показывают «Нет применимых параметров для текущего расчёта».

   **Состояние сворачиваемого блока** (раскрытые/свёрнутые группы) — только для текущей сессии модалки, **не переживает F5** (как и черновик). По умолчанию раскрыты группы с правками или с доступными рычагами; заблокированные — свёрнуты.

   Каждый рычаг внутри группы — это карточка с:
   - заголовком и меткой риска (Низкий / Средний / Высокий риск);
   - редактируемым контролом — выпадающий список для опций (SLA, срок хранения), числовое поле для процентов / целых чисел / дробных;
   - текущим базовым значением «Сейчас: X»;
   - подсказкой «Рекомендуется для уровня: Y» (если ваше значение отличается от рекомендации);
   - диапазоном допустимого — слайдер не может **увеличить** значение выше исходного (это инструмент оптимизации, не повышения);
   - описанием последствия;
   - кнопками `Сбросить параметр` (убрать одну правку из черновика) и `Перейти к полю` (только для рычагов-ответов — открыть соответствующий вопрос Опросника).

5. **Нижняя панель**:
   - `Сбросить изменения` — очищает все правки в черновике (уровень и ограничения остаются).
   - `Применить изменения` — применяет правки к расчёту. **Активна только** при ≥ 1 валидной правке и при отсутствии ошибок в предпросмотре.
   - `Закрыть` — закрывает модалку. Черновик сохранится до F5.

#### Подтверждение изменений с высоким риском

Если среди правок есть **высокорисковые** изменения (снижение целевого уровня SLA, снижение резерва на непредвиденное, снижение запаса на сдвиг расписания), нажатие `Применить изменения` **не применяет их сразу**. Вместо этого внутри той же модалки появляется встроенная панель подтверждения (без отдельного диалога):

```
Подтверждение применения изменений с высоким риском

Вы применяете изменения, которые могут заметно повлиять на надёжность или резервы:
  • Снизить целевой SLA  99.95 → 99.5
    Последствие: Выше допустимое время простоя; ниже требования к резервированию.
  • Снизить непредвиденные расходы  0.05 → 0.02
    Последствие: Меньше подушка на непредвиденное.

[Подтвердить изменения]  [Отмена]
```

- `Подтвердить изменения` запускает реальное применение правок.
- `Отмена` снимает подтверждение; черновик не трогается.

#### Откат последнего применения

После успешного применения в верхней части модалки появляется ярлык:

```
Последнее применение можно откатить, пока модалка открыта.  [Откатить последнее применение]
```

Откат восстанавливает настройки и ответы до состояния перед последним применением (снимок хранится только в памяти модалки) и пересчитывает дашборд. Черновик пересоздаётся свежим на восстановленном расчёте, уровень и тронутые ограничения сохраняются.

Снимок теряется в трёх случаях:
1. F5 (живёт только в текущей сессии).
2. Закрытие модалки → следующее открытие → следующее применение (предыдущий снимок перезаписан).
3. Ручное действие в Опроснике или другом инструменте, изменяющее расчёт (снимок всё ещё доступен через ярлык, но восстановит **только** те поля, которые менял план — изменения других полей сохранятся).

#### Доступные рычаги

| Группа | Поле | Условия |
|---|---|---|
| Инфраструктура и стенды | Размеры стендов НТ / ПСИ / ИФТ / DEV | Разрешение «Можно уменьшать стенды» |
| Риски и резервы | Задачный буфер, проектный буфер, резерв на непредвиденное, запас на сдвиг расписания | Разрешение «Можно снижать риск-буферы» |
| Горизонт планирования | Горизонт планирования (≥ 3 лет) | НЕ на уровне «Консервативный» |
| Надёжность и SLA | Целевой уровень SLA | Разрешение «Можно снижать SLA» И НЕ на уровне «Консервативный»; **высокий риск** |
| Данные и хранение | Срок хранения резервных копий | Разрешение «Можно уменьшать срок хранения»; минимум 90 дней при включённом «Защитить требования соответствия» |
| ИИ / поисковая база | Средний объём ответа модели в токенах | «Большая языковая модель используется» И разрешение «Можно уменьшать ИИ / поисковую базу» |
| ИИ / поисковая база | Размер корпуса поисковой базы, число эмбеддингов | «Поисковая база нужна» И разрешение «Можно уменьшать ИИ / поисковую базу» И НЕ на уровне «Консервативный» |

Значения уровня SLA берутся **из вопроса опросника о целевом SLA** (не зафиксированы в коде) — если вопрос отсутствует или у него нет вариантов, рычаг SLA не появится.

#### Частые вопросы

- **Где увидеть, что я редактирую: расчёт или черновик?** В шапке модалки — уведомление «Изменения сохраняются в черновике и применяются к расчёту только после нажатия „Применить изменения“». Пока вы не нажали «Применить изменения», дашборд / Детализация / PDF показывают **прежнее** состояние.
- **Можно ли применить только часть правок?** Применение применяет все правки в черновике одновременно. Если нужно применить выборочно — сначала снимите ненужные через `Сбросить параметр` на конкретных рычагах, а потом нажмите «Применить изменения».
- **Можно ли применить план дважды?** Нет «плана» как монолита, есть набор ваших правок. После применения черновик очищается (правки уже применены); вы можете внести новые и применить ещё раз. Снимок для отката будет перезаписан.
- **Применяет ли откат правки обратно в черновик?** Нет — откат восстанавливает **расчёт**, не черновик. Черновик после отката — пустой (правок нет, можно начать заново).
- **Что делать, если применение сделало что-то не то, а F5 уже нажат?** Снимок потерян. Восстановить можно либо вручную (зная исходные значения), либо из резервной копии JSON, если она есть. План оптимизации — инструмент для **осознанных** правок; для дорогостоящих экспериментов лучше делать JSON-экспорт перед применением.

### Сводка состояния расчёта

**Дашборд → блок «Сводка состояния расчёта» (сразу под главным блоком)**.

Начиная с версии 2.16, четыре отдельных блока (Готовность / Качество / Бюджет / Следующие шаги) объединены в один сводный блок состояния. Он отвечает на 3 управленческих вопроса:

1. **Можно ли идти к обсуждению?** — главная метка:
   - **Готов к обсуждению** (зелёный) — блокеров нет.
   - **Требует уточнения** (жёлтый) — есть несрочные замечания (бюджет не задан, рискованные допущения, устаревший прайс).
   - **Есть блокеры** (красный) — есть критичные ошибки в качестве, низкое качество (оценка < 60) или расчёт пуст.

2. **Есть ли проблемы по качеству или бюджету?** — две диагностические строки в одну линию:
   - **Качество расчёта** — «Критичных проблем не найдено» / «Есть N критичных проблем» + кнопка «Открыть проверку».
   - **Бюджет** — «CAPEX и OPEX в пределах бюджета» / «CAPEX превышает на +X %» / «Целевой бюджет не задан» + кнопка «Посмотреть рекомендации» (или «Указать бюджет», если целевые суммы не заполнены).

3. **Что делать следующим шагом?** — одна встроенная кнопка основного действия. Берётся первый из активных следующих шагов (сортировка по важности → главное действие). Кнопка ведёт в тот же инструмент, что и раньше: проверка качества / Реестр допущений / Анализ чувствительности (в расширенном режиме) / Бюджетные ограничения / Обоснование расчёта / Мастер уточнения / Импорт прайса / Сравнение сценариев.

Не дублирует проверку качества: модалка «Качество расчёта» (диагностический отчёт по 30+ полям) и сама сводка состояния — это два разных уровня детализации. Сводка — короткая запись о статусе; модалка — полный список замечаний.

### Анализ чувствительности (только в расширенном режиме)

**Боковое меню → переключатель «Расширенные настройки» → Дашборд → блок «Следующие шаги» → подсказка «Найти драйверы стоимости»**.

Основной анализ стоимости находится на Дашборде:
- **Распределение по категориям** — структура затрат (что сейчас дороже всего: Услуги / Лицензии / Аппаратные ресурсы / Безопасность / Трафик).
- **Вклад риск-коэффициентов** — сколько добавила наценка (буферы / инфляция / резерв / сдвиг расписания) и в чём её источник.

Для большинства задач этих двух блоков достаточно — они прямо отвечают на вопросы «что дорого?» и «сколько добавили риски?».

Анализ чувствительности — расширенный инструмент возмущений для архитектора: для каждого числового параметра он временно изменяет его на ±10 % (переключатели — переключает) и пересчитывает итог, ранжируя поля по абсолютной дельте. Отвечает на другой вопрос: **«какую вводную выгоднее изменить, чтобы снизить итог»**. Категория может быть огромной, но неэластичной (фиксированная позиция прайса) — анализ чувствительности находит маленькие, но эластичные рычаги.

Доступен только при включённом «Расширенном режиме» — иначе подсказка «Найти драйверы стоимости» в «Следующих шагах» скрыта.

### Бюджетные ограничения

**Дашборд → блок «Бюджет» в главном блоке → клик**.

Сравнение CAPEX и OPEX вашего расчёта с целевыми значениями (вопросы «Целевой CAPEX, ₽» и «Целевой OPEX в месяц, ₽» в Опроснике). Показывает превышение / запас, рекомендации.

### Обоснование расчёта

**Дашборд → блок «Следующие шаги» → «Сформировать обоснование расчёта»** (также доступно из проверки качества и Бюджетных ограничений).

Собирает управленческое обоснование расчёта в формате Markdown:
- название расчёта, активный сценарий, провайдер и версия прайса;
- итоговые CAPEX и OPEX;
- оценка качества из проверки качества;
- источники ответов, рискованные допущения;
- ошибки и предупреждения;
- топ-5 драйверов стоимости;
- статус относительно бюджета;
- рекомендации.

**Действия**: «Скопировать Markdown» (в буфер обмена) или «Скачать .md». Обоснование собирается на лету, ничего не сохраняется и не отправляется наружу.

## Следующий шаг (внутри «Сводки состояния расчёта»)

Начиная с версии 2.16, отдельной карточки «Следующие шаги» нет — её заменила **встроенная кнопка основного действия** внутри «Сводки состояния расчёта». Это сделано осознанно: четыре карточки дашборда, описывающие одно и то же состояние, конкурировали между собой и заставляли собирать общую картину из разрозненных блоков.

**Это не список изменений, которые приложение применит автоматически.** Кнопка «Следующего шага» открывает существующий инструмент — проверку качества, Реестр допущений, Анализ чувствительности, Бюджетные ограничения, обоснование расчёта, Мастер уточнения, Импорт прайса, Сравнение расчётов. Никакого применения, предпросмотра, отката, изменения сценария.

### Какое действие подскажет блок

Берётся первое из активных следующих шагов, отсортированных по важности:

- **Уточнить критичные вводные** — Мастер уточнения, если в расчёте есть ошибки качества или оценка < 60.
- **Просмотреть предупреждения** — проверка качества при ≥ 3 предупреждениях.
- **Проверить рискованные допущения** — Реестр допущений при ≥ 3 рискованных значениях по умолчанию.
- **Изучить превышение бюджета** — Бюджетные ограничения при превышении CAPEX/OPEX. Сопутствующая подсказка **«Найти драйверы стоимости»** (Анализ чувствительности) появляется только в расширенном режиме.
- **Обновить прайс провайдера** — Импорт прайса, если применённый прайс заметно отстаёт от поставляемого с приложением.
- **Сравнить сценарии** — Сравнение сценариев при ≥ 2 сценариях.
- **Сформировать обоснование расчёта** — обоснование расчёта, когда расчёт выглядит готовым.

Полный список рекомендаций (если их больше одной) доступен из тех же инструментов — модалка проверки качества и Реестр допущений показывают свой набор подсказок, Бюджетные ограничения — свой. Дашборд показывает только **главное следующее действие**, чтобы не дробить экран.

### Важность и сортировка

- **Критично** — красная полоса (ошибки / превышение бюджета).
- **Важно** — оранжевая (низкая оценка / рискованные допущения / устаревший прайс).
- **Полезно** — зелёная (множество сценариев — есть что сравнить).
- **Информационно** — серая (обоснование как фоновое действие).

На одну цель — одна кнопка на Дашборде (без дублирования).

## Прайс

Прайс — это таблица цен ЭК для активного провайдера. Обновляется только через **Импорт прайса JSON**. Автоматического обновления с сайтов провайдеров нет (см. ниже).

### Импорт прайса JSON

**Опросник → блок «Тарифы провайдера» → Импорт прайса**.

Окно выбора файла открывает мастер импорта. Поддерживаемые форматы:
- **JSON-файл провайдера** в формате версии 1 — мастер пропускает сопоставление и сразу проверяет.
- **CSV** (запятая, точка с запятой, табуляция; UTF-8 с маркером последовательности или без). Распознаются заголовки на русском и английском.
- **JSON-массив** объектов: `[{ "service": "vCPU shared", "price": 920, "unit": "month" }, ...]`.

**Шаги мастера**:

1. Выбрать провайдера (по умолчанию — текущий из настроек расчёта) и нажать «Выбрать CSV/JSON файл».
2. **Предпросмотр** первых 8 строк с автоматическим сопоставлением.
3. **Сопоставление** — каждая строка → выпадающий список ЭК. Уровни уверенности:
   - **высокая** — точное совпадение идентификатора или известный синоним.
   - **средняя** — пересечение слов между названием в файле и названием ЭК ≥ 50%.
   - **низкая** — пересечение слов 30–50% (требует подтверждения).
   - **нет** — авто-сопоставление не предложено; нужно выбрать вручную.
4. **Проверка** — сводка: сопоставлено / не сопоставлено / ошибки. Кнопка **Применить прайс** активна только если ошибок нет.

**Что мастер проверяет**:
- Цена — конечное положительное число (`14 400,50` → 14400.5).
- Каждая строка сопоставлена с существующим ЭК.
- Нет двух строк, сопоставленных с одним и тем же ЭК.
- В файле не более 1000 строк (защита от случайной загрузки 10 МБ CSV).

Применённый прайс уходит в стек откатов — в стек попадает прежний прайс, можно откатить через «Откатить на прайс <версия>».

### Жизненный цикл применённого прайса

1. Применяете новый прайс — старый снимок уходит в **стек откатов** (лимит 3).
2. Каждый расчёт хранит метку «какой прайс применён».
3. Если в браузере появляется более свежий прайс, чем у активного расчёта — над ним появляется индикатор **«Старый прайс»** + кнопка **«Пересчитать на новом прайсе»**.
4. Применение — атомарное: либо все ЭК расчёта получают новые цены, либо ничего.

### История прайсов

**Опросник → блок «Тарифы провайдера» → История**.

Модалка показывает:
- **Текущий** — применённый сейчас прайс (источник, версия, время применения).
- **Предыдущие 1..3** — последние снимки в стеке откатов.

Для каждой записи:
- Версия и время применения.
- Топ-3 `↑` (что подорожало) и `↓` (что подешевело) — метка дельты со всплывающей подсказкой «Цена за единицу X → Y ₽».
- Кнопка **Откатить на эту точку**.

Откат на точку N работает так: все более новые точки удаляются вместе с текущей; целевая становится текущей; точки старше цели сохраняются как «прошлое».

### Прайс-бенчмарк

**Опросник → блок «Тарифы провайдера» → Прайс-бенчмарк**.

Сравнение цен провайдеров (только для просмотра): таблица 4 категории (CPU / RAM / STORAGE / NETWORK) × все провайдеры. Каждая ячейка — представительная цена категории (ключевой элемент: общее виртуальное ядро для CPU, гигабайт памяти для RAM, и т. д.).

В каждой категории самый дорогой ЭК подсвечен. Помогает быстро увидеть «у кого дороже за единицу». **Не путайте с Анализом чувствительности**: самый дорогой ЭК — это просто цена за единицу, чувствительность — влияние на итоговый бюджет.

Сортировка: клик на заголовок категории → по возрастанию/убыванию по этой колонке. По умолчанию — по «Итого» по возрастанию.

Метка дельты показывается, если эффективная цена отличается от поставляемой по умолчанию (т. е. провайдер имеет применённый прайс).

### Сводка по провайдеру (в Опроснике)

Под блоком выбора провайдера в Опроснике — сводка по 6 категориям активного провайдера: CPU / RAM / Хранилище / Сеть / Лицензии / Услуги. Для каждой категории:
- название (с иконкой);
- список ЭК с ценой за единицу в рублях;
- самый дорогой ЭК подсвечен;
- если есть применённый прайс — метка дельты рядом с ценой.

### Почему нет автоматического обновления

Калькулятор работает строго локально и не делает сетевых запросов к сайтам провайдеров. Все цены поставляются вместе с приложением в виде JSON-файлов. Кратко — это сделано из соображений безопасности (политика безопасности контента запрещает внешние запросы), отсутствия публичных API биллинга и хрупкости разбора HTML. Полное обоснование и регламент квартального обновления цен — в [MAINTAINER_GUIDE.md](MAINTAINER_GUIDE.md).

### PDF

Кнопка **PDF** в верхней панели печатает то, что сейчас открыто:
- **Опросник** → таблица «Вопрос → Ответ» (опционально + «Пояснение» в расширенном режиме). Через модалку «Параметры PDF» можно выбрать книжную или альбомную ориентацию.
- **Дашборд / Сравнение** → текущий вид со сбросом цветов на белый фон; тёмная и светлая темы корректно сбрасываются для печати.
- **Детализация** → A4 альбомная ориентация с малыми колонками. На бумаге скрыты «Доля %», «Риск %», «Риск ₽» (производные от ИТОГО).

### JSON: один расчёт

**Расчёты → ⋯ → Экспорт JSON** или `Ctrl+Alt+S` для активного. JSON содержит настройки, ответы, снимок каталога ЭК, список вопросов, настройки представления, метку версии прайса. Импорт — **Расчёты → Импорт JSON**. При совпадении идентификаторов появится модалка «Заменить / Создать копию».

Кнопки `Импорт JSON` / `Экспорт JSON` в верхней панели — быстрый доступ к тем же действиям для активного расчёта.

### JSON: полное состояние (полный пакет)

**Расчёты → Полный экспорт** — выгружает все расчёты + общий справочник одним файлом полного пакета. Используйте для резервной копии перед обновлением каталога ЭК или сменой браузера.

### CSV-детализация

В **Детализация** → **Скачать CSV**. Колонки: ЭК × стенд × категории. Кодировка UTF-8 + маркер последовательности, разделитель `;`, десятичный `,`. Защита от подмены команд в CSV: ячейки с `=`/`+`/`-`/`@`/`\t`/`\r` экранируются префиксом `'`.

### Обоснование расчёта

См. раздел [Анализ расчёта → Обоснование расчёта](#обоснование-расчёта).

## Расширенные настройки (Администрирование)

По умолчанию приложение показывает только обычный пользовательский интерфейс. Переключатель **«Расширенные настройки»** (в нижней части бокового меню, рядом со «Справкой») открывает группу **«Администрирование»** с двумя разделами для модификации модели расчёта. Сам переключатель всегда виден; группа «Администрирование» появляется только при включении. Состояние переключателя переживает F5 (сохраняется в браузере).

Горячие клавиши `Ctrl+Alt+6` (Элементы) и `Ctrl+Alt+7` (Вопросы) подчиняются тому же ограничению: вне расширенного режима они показывают подсказку «Включите Расширенные настройки», но не открывают вкладку.

### Элементы

Управление каталогом ЭК (создание / чтение / обновление / удаление; 36 позиций, включая 7 позиций для ИИ):
- Создать / редактировать / удалить элемент.
- Поправить цену, единицу, категорию, формулу количества.
- JSON-импорт / экспорт каталога.
- **Цены CSV** (экспорт / импорт) — массовое обновление цен по идентификатору. Подробности формата: см. ниже.

#### Цены CSV (для администраторов)

Формат:
```
id;name;vendor;unit;category;resourceClass;billingInterval;pricePerUnit;priceUpdatedAt;priceSource
```

Импорт принимает любой CSV с минимум двумя колонками — идентификатор и цена за единицу. Автоопределение разделителя (`;` или `,`). Десятичный — `.` или `,`. Сопоставление по идентификатору, обновляется только цена за единицу.

Защита от опечаток:
- **Корректность** — конечное число.
- **Граничные** — 0 ≤ цена ≤ 10¹².
- **Аномалии** — если новая цена ≥ 10× старой ИЛИ ≤ старая/10, в отчёте предупреждение «возможна опечатка».

### Вопросы

Управление справочником вопросов опросника (создание / чтение / обновление / удаление):
- Создать / редактировать / удалить вопрос.
- Поправить тип, опции, условия зависимости, текст подсказки.
- JSON-импорт / экспорт.

Используйте только если действительно нужно изменить модель расчёта под конкретный продукт. Для типового использования достаточно ответов в Опроснике без правки справочников.

## Устранение неполадок

### Браузер кэширует старую версию

После обновления приложения вы видите старые названия ЭК / тексты / интерфейс?

**Решение**: полное обновление страницы — **Ctrl+Shift+R** (Windows/Linux) или **Cmd+Shift+R** (macOS). Браузер агрессивно кэширует модули; обычный F5 не всегда очищает.

### CSV/JSON не загружается — «Файл не является корректным JSON»

Проверьте кодировку (должна быть UTF-8) и парность скобок. Самая частая ошибка — экспорт из Excel с кодировкой ANSI / Windows-1251. Откройте файл в Notepad++ → Кодировка → Преобразовать в UTF-8 → сохранить.

### CSV-импорт цен помечает все строки как аномалии

Возможно, неверный десятичный разделитель — приложение видит `1,500` (рус.) как 1500 рублей вместо 1.5. Используйте экспорт текущих цен из приложения как **шаблон** — оно генерирует CSV с корректным форматом.

### Расчёт даёт 0 ₽ по всему

Скорее всего, в Опроснике не заполнены ключевые нагрузочные параметры: «Пиковая нагрузка в запросах в секунду», «Количество микросервисов», «Количество баз данных». Большинство ЭК зависят от этих параметров — при нулях формула количества тоже даёт 0.

**Решение**: пройдитесь по Опроснику и заполните 5–7 ключевых нагрузочных параметров.

### Всплывающая подсказка пропадает / не появляется

Всплывающая подсказка — это стандартный механизм браузера: текст появляется через 1–2 секунды после наведения мыши. Задержку контролирует браузер, мы её не настраиваем.

Для информационной кнопки (с иконкой `?` или `i`) — клик открывает модалку с полным описанием.

### Данные пропали после закрытия браузера

Причины:
- **Режим инкогнито/приватный** — данные в браузере очищаются при закрытии. Используйте обычное окно.
- **Очистка куки / данных сайта** — браузер удалил вместе с другими данными. Восстановление возможно только из резервной копии JSON.

Профилактика: периодически делайте **Расчёты → Полный экспорт**, особенно перед обновлением приложения или сменой браузера.

### «Прайс обновляется в другой вкладке»

Защита от параллельных вкладок: вы открыли несколько вкладок калькулятора, и в одной из них уже идёт применение того же провайдера. Подождите, пока операция там завершится, либо закройте ту вкладку. Блокировка снимается автоматически через 60 секунд.

### «Приложение уже открыто» во второй вкладке/окне

Калькулятор разрешает работать **только в одном экземпляре одновременно** в одном браузере. Это сделано намеренно — две одновременно открытые вкладки конкурируют за общее хранилище расчётов, последняя запись затирает предыдущую, и сохранённые расчёты могут «исчезнуть».

При попытке открыть вторую вкладку/окно вы увидите экран «Приложение уже открыто» с инструкцией и одной кнопкой **«Проверить снова»**:

- Закройте лишнюю вкладку нативной горячей клавишей браузера — **Ctrl+W** (Windows/Linux) или **Cmd+W** (macOS).
- В оставшемся окне нажмите **«Проверить снова»**, если оно тоже было заблокировано (например, обе вкладки открыты до того, как одна успела захватить защиту) — после закрытия другой страница перезагрузится и приложение откроется нормально.

Если первое окно крашнулось / закрылось без cleanup'а — защита сама отпустит блокировку через 90 секунд (приложение перестанет считать предыдущий экземпляр живым).

Если первое окно крашнулось / закрылось без cleanup'а — защита сама отпустит блокировку через 90 секунд (приложение перестанет считать предыдущий экземпляр живым).

**Ограничения защиты**: блокировка действует **только в пределах одного браузера на одном компьютере**. Разные браузеры (Chrome + Edge), разные профили одного браузера, обычное окно + Инкогнито, разные пути запуска (`file://` vs `http://localhost:8000`) **не видят друг друга** — между ними блокировка не работает, потому что у каждого из них собственное хранилище. Используйте только один способ запуска для работы с реальными данными.

### Опросник: длинная анкета, сложно ориентироваться

- Используйте **сворачиваемые секции** (раскрыть/свернуть слева от названия).
- **Индикатор прогресса** в заголовке каждой секции показывает «N / M уточнено».
- **Подгруппы** (поисковая база, большая языковая модель, персональные данные, настройка модели) тоже сворачиваемые.
- Состояние «что раскрыто» сохраняется в браузере — после F5 вернётесь туда же.

### Поле неактивно, не могу изменить

Это **зависимое от главного переключателя поле**. Над/рядом с ним должен быть переключатель, который его открывает. Всплывающая подсказка объясняет: «Поле неактивно: сначала включите …».

Например, поля для поисковой базы активируются после включения переключателя «Поисковая база нужна».

### Поле зачёркнуто в опроснике

Это неактивное или приглушённое поле — обычно потому, что включён режим «Без рисков». Параметры рисков (буферы, инфляция, сезонность, расписание, непредвиденные) тогда не применяются и приглушаются.

## Импорт прайсов провайдера и НДС (с версии 2.16.0)

С версии 2.16.0 калькулятор различает цены **с НДС** и **без НДС** на уровне JSON-файла провайдера.

### Что важно

- **Внутри расчёта цены всегда хранятся без НДС.** НДС применяется отдельно по настройкам расчёта (Опросник → блок «НДС»). Это значит, что итог НЕ может оказаться завышенным из-за двойного учёта НДС.
- В блоке «Тарифы активного провайдера» появилась поясняющая метка: «Цены сохранены без НДС. НДС применяется отдельно в расчёте.»
- Если источник прайса содержал цены С НДС — это видно в дополнительной строке: «Источник содержал цены с НДС. При импорте они сохранены как цены без НДС.»

### Импорт нового JSON

Если вы импортируете провайдерский JSON современного формата (с явной политикой НДС в файле), приложение применяет цены автоматически.

Если вы импортируете старый JSON-файл (без явного указания политики НДС), приложение спросит:
- **Цены без НДС** — цены применяются как есть (без НДС);
- **Цены с НДС 20%** — цены делятся на 1.20 (период 2019–2025);
- **Цены с НДС 22%** — цены делятся на 1.22 (с 01.01.2026);
- **Отмена** — импорт не выполняется, прайс не меняется.

По умолчанию вариант **не выбран** — вы должны явно указать, что в файле, чтобы избежать ошибки. Текстовая пометка «с НДС» в описании прайса не используется как подсказка — выбор всегда делает пользователь.

### Старые сохранённые расчёты

При открытии расчёта, созданного до версии 2.16.0, может появиться напоминание: «Старые расчёты могли учитывать НДС дважды. Проверьте применённый прайс…»

Расчёт **не меняется автоматически**. Если хотите пересчитать на актуальном прайсе:
1. Откройте блок «Тарифы активного провайдера» (кнопка «Перейти к тарифам» в напоминании).
2. Импортируйте JSON провайдера через выбор файла — современный формат применится без вопросов; старый — попросит выбрать политику НДС.
3. Нажмите «Пересчитать на новом прайсе» — текущий расчёт обновится с новыми ценами и тем НДС, что задан в Опроснике.

### Если провайдер — VK Cloud

Цены VK Cloud в текущем встроенном комплекте помечены как **«НДС-политика принята по допущению»**. Это означает, что цены — реалистичная заглушка (не верифицированный публичный прайс), и они трактуются как цены с НДС 22%. Когда появится верифицированный источник, метка обновится автоматически.
