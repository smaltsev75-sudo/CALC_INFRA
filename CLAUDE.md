## 🎨 UI/UX Review Standard для локальных ванильных веб-приложений

Этот шаблон зафиксирован 2026-05-05 как обязательный формат отчёта при любом запросе на **визуальный аудит / UI/UX-ревью / вёрстку / рефакторинг CSS / создание новых экранов** на vanilla-стеке. Применяется без дополнительных уточнений и **дополняет** общий код-ревью стандарт ниже.

### Шкала оценки UI 1–10

- **1–3**: Визуальный хаос, отсутствие системы, критические проблемы с адаптивом/a11y, UX мешает использованию или провоцирует ошибки.
- **4–6**: Базовая работоспособность, но нет дизайн-системы, инконсистентные отступы/цвета/типографика, слабая отзывчивость, ручные стили без токенов.
- **7–8**: Чёткая визуальная иерархия, адаптивность, кастомные свойства, предсказуемые состояния интерфейса, есть места для полировки и оптимизации рендеринга.
- **9–10**: Production-ready UI, полная дизайн-система, безупречный a11y, оптимизированный paint/layout, готовые паттерны масштабирования и темизации.

### Критерии аудита (8 осей)

1. **Визуальная система и консистентность** — токены `--color-*`/`--space-*`/`--font-*`, единая сетка, темизация через `[data-theme]`, унифицированные иконки.
2. **Адаптивность и отзывчивость** — breakpoints, fluid layouts через `clamp()`, container queries (`@container`), mobile-first/desktop-first стратегия, touch-target для coarse pointer.
3. **UX-потоки и состояния интерфейса** — `:hover`/`:focus`/`:focus-visible`/`:active`/`:disabled`, loading/error/success states, микроанимации, `prefers-reduced-motion`, persist UI-state.
4. **Доступность (a11y) и семантика** — контраст WCAG AA/AAA (≥4.5:1 normal text, ≥3:1 large text/UI), `:focus-visible`, ARIA-атрибуты (`aria-labelledby`/`aria-pressed`/`aria-invalid`/`aria-live`), skip-link, клавиатурная навигация, screen-reader.
5. **Архитектура CSS и производительность** — специфичность, `@layer` для управления каскадом, отсутствие `!important`-loop, `contain` containment, `will-change` точечно (без overuse), `content-visibility: auto` для скрытых аккордеонов, paint/layout thrashing.
6. **Компонентная структура и переиспользование** — нативные Web Components / `customElements.define` (если применимо), CSS-изоляция через namespace или Shadow DOM, темизация через `var()`, slot-based API.
7. **Визуальная иерархия и читаемость** — масштаб типографики, whitespace, выравнивание, группировка данных, визуальный вес, `tabular-nums` для денежных колонок, `max-width + ellipsis` для длинного текста.
8. **Прогрессивное улучшение и фолбэки** — `@supports` для современных свойств (backdrop-filter, `:has()`, `@container`), graceful degradation, корректное отображение при отключении JS / CSS-анимаций, `prefers-color-scheme` / `prefers-contrast` фолбэки.

### Ограничения шаблона UI/UX

- Анализировать ТОЛЬКО текущую кодовую базу. Не выдумывать дизайн-системы, палитры, зависимости.
- Фокусироваться на нативных HTML/CSS/JS решениях. **НЕ предлагать**: Tailwind, CSS-in-JS, препроцессоры (Sass/Less), UI-библиотеки.
- Учитывать специфику предметной области (для калькулятора: плотность данных, читаемость таблиц/форм, инпуты, визуализация результатов, состояния валидации).
- Если не хватает контекста — задать **максимум 1** вопрос, но всё равно выдать отчёт по имеющемуся коду.

### Vanilla UI/UX чеклист (быстрая проверка)

При выдаче ревью убедиться, что охвачены:
- Используются ли `clamp()` для fluid typography (vs hardcoded rem на breakpoints)?
- Есть ли `@layer` для управления каскадом (cascade-через-порядок-импортов хрупок)?
- Используются ли `@container` для компонентной адаптивности (vs только `@media` viewport)?
- `:focus-visible` split mouse/keyboard (focus-ring только для keyboard)?
- Touch-target ≥44×44 на `(pointer: coarse)`?
- `prefers-reduced-motion` обнуляет animations/transitions?
- `[aria-invalid="true"]` на полях с ошибкой валидации + inline-сообщение через `aria-describedby` (не только snackbar)?
- `@supports` фолбэк для `backdrop-filter` / `:has()` / `@container`?
- `content-visibility: auto` на скрытых аккордеонах для GPU-экономии?
- Темизация через `[data-theme]` + `var()` (не через JS-class swap каждой палитры)?
- `tabular-nums` на денежных колонках (выравнивание разрядов)?
- `aria-live` на статус-сообщениях (persist-indicator, snackbar)?
- Контраст: `--text` на `--bg-card` ≥4.5:1, `--accent` на `--bg-card` ≥3:1 (UI-component) для обеих тем?
- Нет ли дубля одной и той же информации в видимой строке, tooltip'е, бейдже
  или соседнем тексте? Дубли даты/статуса/значения не помогают пользователю,
  а рассеивают внимание. Если tooltip повторяет видимый текст — tooltip убрать.
  Если общий блок уже сообщает статус для всего scope — не повторять тот же
  статус на каждой строке; показывать только отличия и исключения.

## Архитектура: 6 слоёв, односторонний поток

```
ui/          — DOM рендер (без бизнес-логики)
controllers/ — events → store (оркестровка)
state/       — store (deepFreeze + revision), persistence, migrations
services/    — IO (storage, json, csv, markdown, jsonImport)
domain/      — чистая логика расчёта и DSL (без DOM, без store)
utils/       — низкоуровневые примитивы (constants, escapeHtml, debounce, lru, freeze, uuid)
```

**Поток данных**: `event → controller → store.update() → notify subscribers → rAF render + persist`. Подписчик в [js/app.js](js/app.js) планирует ре-рендер через `requestAnimationFrame` — никогда не вызывайте `renderApp` напрямую из контроллеров.

**Правила слоёв**:
- `domain/` не знает о DOM и store. Принимает `(calc, settings, ...)` → возвращает чистые данные. Тесты `tests/unit/domain/*` — именно сюда.
- `controllers/` — единственное место, где UI-события превращаются в `store.update*()` и вызовы `services/*`. Ловят ошибки и пробрасывают в snackbar.
- `ui/` собирает DOM из state через `el()`/`replace()` (см. [js/ui/dom.js](js/ui/dom.js)). Никогда не мутирует state напрямую — только через `ctx`-обёртки, передаваемые из [js/app.js](js/app.js).
  - **Layout** (после Этапа 9.6): левый sidebar ([js/ui/sidebar.js](js/ui/sidebar.js), 220px → 64px на ≤1100px) + topbar ([js/ui/header.js](js/ui/header.js)). Файл `js/ui/tabs.js` удалён.
  - **Иконки** ([js/ui/icons.js](js/ui/icons.js)) — inline SVG из библиотеки **Lucide** (ISC, copy-paste). API: `icon(name, { size })`. **Эмодзи в UI запрещены**, только в комментариях кода.
  - **info-иконка** унифицирована — один CSS-класс `.info-icon`. Поведение по тегу: `<button>` — clickable + pointer; `<span>` — tooltip-only + help. `infoIcon()` helper в [js/ui/dom.js](js/ui/dom.js).
  - **Тема приложения** (12.U33) — UI-параметр, `state.ui.theme: 'dark'|'light'`, persist в `STORAGE_KEYS.THEME = 'calc.theme'`, restore в `calcListController.initFromStorage`. Дефолт = `DEFAULT_THEME = 'dark'`. НЕ входит в JSON-экспорт расчёта. Контроллер: [calcController.setTheme(theme)](js/controllers/calcController.js) (валидирует) + `toggleTheme()`. Применение: `applyThemeAttribute(theme)` в [app.js](js/app.js) ставит `<html data-theme="light">` или удаляет атрибут для default. Вызывается **на boot ДО первого рендера** (избегает flash-of-wrong-theme при F5 в light) + в subscriber. Светлая палитра в [base.css](css/base.css) `[data-theme="light"]`: `--bg-card: #ffffff`, `--text: #0f172a`, `--accent: #059669` (emerald-600, ~4.5:1 на белом — `#26d49a` даёт ~2:1, WCAG fail). Кнопка переключения — Sun/Moon в topbar, текст «куда переключим». Тесты: [theme-persist.test.js](tests/unit/state/theme-persist.test.js), [theme-controller.test.js](tests/unit/controllers/theme-controller.test.js).
- `services/` — побочные эффекты и IO. Возвращают `{ ok, reason, ... }`-объекты, а не бросают исключения для ожидаемых ошибок (отмена пользователем, валидация).

## Store: иммутабельность через deepFreeze

[js/state/store.js](js/state/store.js) хранит весь state как **глубоко замороженный** объект. Любая «мутация» = создание нового объекта и пере-`deepFreeze`. Если что-то внезапно бросает `TypeError: Cannot assign to read only property` — это сигнал, что код пытается мутировать state напрямую. Решение: использовать `store.updateActiveCalc({ ... })`, а не `state.activeCalc.foo = ...`.

`calcRevision` инкрементируется при каждом изменении активного расчёта; используется как ключ кэша для `calculate()` (LRU в [js/utils/lru.js](js/utils/lru.js)).

### Persist UI state

UI-настройки переживают F5 — это требование ([memory/feedback_persist_ui_state.md](C:/Users/Сергей/.claude/projects/d--DATA------------------------------------------/memory/feedback_persist_ui_state.md)). Что сохраняется:
- `STORAGE_KEYS.ACTIVE_TAB = 'calc.activeTab'` — последняя открытая вкладка ([js/state/persistence.js](js/state/persistence.js) → `loadActiveTab()`/`saveActiveTab(id)`).

## Расчёт стоимости — где живёт формула

Системная формула собирается в [js/domain/calculator.js](js/domain/calculator.js):

```
costFinal  = costBase × (applyRisks ? riskTotal : 1) × vatMul        // 12.U20
costBase   = qty × pricePerUnit × billingIntervalMul
riskTotal  = bufferFactor × inflationMul × seasonalMul × scheduleMul × contingencyMul   // БЕЗ vatMul
vatMul     = vatEnabled ? (1 + vatRate) : 1                          // независимая ось
```

`applyRisks = calc.settings.applyRiskFactors` (boolean, default `true`). При `false` риск-наценка не накапливается в итог (`costFinal = costBase × vatMul`), но `cell.riskBreakdown` ВСЕГДА содержит реальные коэффициенты — это позволяет UI показать «потенциальную» наценку даже в режиме «без рисков». Переключатель живёт в Опроснике (блок «Параметры расчёта»). При выключении 7 связанных полей (буферы, инфляция, сезонность, расписание, непредвиденные, горизонт планирования) становятся `disabled`.

**12.U20: НДС — независимая ось от риск-коэффициентов.** НДС не «риск», а налог: пользователь либо учитывает его в бюджете, либо нет, и это решение никак не связано с тем, накручиваем ли мы буферы/инфляцию/сезонность сверху. Подгруппа «НДС» в Опроснике НЕ блокируется при выключенном мастере рисков, и НДС применяется к итогу даже когда `applyRiskFactors=false`. `riskBreakdown.total` теперь содержит ТОЛЬКО риски (без `vatMul`); `vatMul` остаётся в breakdown как информация для tooltip'ов. Карточка «Вклад риск-коэффициентов» на дашборде больше не включает НДС в список компонентов.

Где какие коэффициенты применяются — задокументировано в шапке `calculator.js` и продублировано в `ctx.openSummaryFormula()` ([js/app.js](js/app.js)). При изменении модели коэффициентов **обновлять оба места**.

`qty` для каждой пары (item × stand) считается через **собственный безопасный DSL** — НЕ через `eval`/`new Function`:
- Парсер: [js/domain/formula/parser.js](js/domain/formula/parser.js) (рекурсивный спуск, AST).
- Кэш AST: [js/domain/formula/cache.js](js/domain/formula/cache.js).
- Вычислитель: [js/domain/formula/evaluator.js](js/domain/formula/evaluator.js). Whitelist встроенных функций: `min/max/round/ceil/floor/abs/clamp/if`.
- Идентификаторы:
  - `Q.<id>` — ответ на вопрос (одноуровневый, плоский map; `Q.foo.bar` — синтаксическая ошибка).
  - `S.<param>` или `S.<param>.<sub>...` — настройка расчёта; **многоуровневый dot-доступ** для вложенных объектов (например, `S.standSizeRatio.DEV`). AST-узел `Var` хранит сегменты в `path: string[]`.

Линтер формул в [js/domain/validation.js](js/domain/validation.js) находит висящие ссылки `Q.<id>` на отсутствующие вопросы. Вызывается при импорте расчёта и при удалении вопросов.

## Миграции схемы

[js/state/migrations.js](js/state/migrations.js) содержит массив `MIGRATIONS` шагов `from → to`. Текущая версия вычисляется автоматически из последнего шага `MIGRATIONS`; на PATCH 2.20.39 это **20** (см. `LATEST_SCHEMA_VERSION` в [js/state/migrations.js](js/state/migrations.js) и re-export `CURRENT_SCHEMA_VERSION` в [js/utils/constants.js](js/utils/constants.js)). Не дублировать номер схемы в production-коде.

При добавлении миграции:
1. Реализовать `step.run(calc)` — мутирует **глубокую копию**, не оригинал.
2. Шаг должен быть **идемпотентным**.
3. Добавить запись в `MIGRATIONS` в порядке возрастания `to`.
4. Добавить тесты в [tests/unit/state/migrations.test.js](tests/unit/state/migrations.test.js).

Миграции применяются и при загрузке из localStorage, и при импорте JSON. Bundle-импорт ([js/services/bundleExport.js](js/services/bundleExport.js)) делает atomic backup/rollback — если хоть один расчёт не мигрирует, состояние не меняется.

## Версионирование приложения (SemVer)

`APP_VERSION` ([constants.js:11](js/utils/constants.js#L11)) и `package.json` "version" — два источника правды, синхронизируются вручную. Линтер [app-version-sync.test.js](tests/unit/architecture/app-version-sync.test.js) ловит расхождение в CI.

Когда bump'ить:

| Часть | Когда инкрементируется | Примеры |
|---|---|---|
| **MAJOR** (`X.0.0`) | Breaking change для пользователя или формата bundle: удалена вкладка/функция, изменён JSON-формат (= bump `BUNDLE_MAJOR`), несовместимый UX | Снос Сравнения; смена структуры bundle-файла |
| **MINOR** (`1.X.0`) | Крупный этап / новая видимая фича / новый persistent-контракт расчёта | UI-редизайн 9.6, светлая тема 12.U33, AI-агенты 13, инвариант стенд ≤ ПРОМ 13.U11 |
| **PATCH** (`1.1.X`) | Багфиксы, рефакторинг, мелкие UX-правки, hardening, обновления прайсов без новых фич, compatibility-миграции без нового формата данных | 12.U30, 12.U31, 12.U34 (cleanup pass), 2.20.20 Quick Start normalization |

**Правило bump'а одной строкой**: новый пользовательский persistent-контракт или видимая фича → MINOR; compatibility-миграция, которая только чинит/нормализует старые данные без нового формата, может быть PATCH; ломаем формат bundle → MAJOR.

Bump делается **синхронно в двух файлах** (`constants.js` + `package.json`) одним коммитом — линтер не пропустит рассогласованное состояние.

### DOM/CSS

- **`<select>`**: устанавливать `value` ПОСЛЕ добавления `<option>` детей.
- **`el('option', { value })` ставит value через IDL** (12.U21, [dom.js](js/ui/dom.js)). При добавлении нового тега в whitelist `el()` НЕ забывать про IDL-properties — `setAttribute('value', x)` отражается не для всех тегов. Тест: [dom-option-value.test.js](tests/unit/ui/dom-option-value.test.js).
- **CSS-анимация на пересоздаваемом узле = flicker** (12.U22). Любая `animation` на overlay/модалке должна жить на классе-маркере «свежесть» (`.modal-overlay-fresh`), не на базовом селекторе. Тот же паттерн что `.tab-pane-fresh`. Тесты: [modal-flicker.test.js](tests/unit/ui/modal-flicker.test.js).
- **`flex-wrap` НЕ даёт детерминированного layout** (12.U24-fix-3). При узкой шапке с переменной длиной subtitle описывай структуру явно через `flex-direction: column` + 2 обёртки (`header-top` + `badges`). Тесты: [stand-card-badges.test.js](tests/unit/ui/stand-card-badges.test.js).
- **`flex-wrap: wrap` + фиксированная `height` — анти-паттерн** (12.U3). Использовать `min-height` + дочерние получают свою высоту.
- **Любой `overflow ≠ visible` на ancestor таблицы ломает sticky-thead** (12.U30 1.4c+1.5e, 12.U31 A.1). По CSS-spec `overflow-x: auto` авто-задаёт `overflow-y: auto` → создаёт scroll-context → sticky привязывается к ancestor, не к viewport. **Жертв в проекте уже три**: `.app-main`, `.items-table-wrap`, `.comparison-table-wrap`. Линтеры: [items-wrap-overflow.test.js](tests/unit/ui/items-wrap-overflow.test.js), [comparison-wrap-overflow.test.js](tests/unit/ui/comparison-wrap-overflow.test.js), [sticky-thead-positioning.test.js](tests/unit/ui/sticky-thead-positioning.test.js). При создании нового wrap'а — НЕ задавать overflow вообще. **Исключение**: мини-таблицы БЕЗ sticky-thead (например `.comparison-ai-block-table-wrap` в Сравнении/AI-метрики, 13.U11) — на них `overflow-x: auto` безопасен и иногда нужен как страховка от экстремальных значений.
- **`min-width: 0` на grid-items с табличным контентом внутри** (13.U11). Дефолтный `min-width: auto` для grid-item означает, что трек растёт по min-content ребёнка. Если ребёнок — таблица с `white-space: nowrap`, min-content = ширина самой длинной строки. Это игнорирует `1fr`-аллокацию трека, и item пролезает в соседнюю карточку или за viewport. Симптом: «карточка визуально лопается, числа вылезают за рамку». Фикс: `min-width: 0` на grid-item. Жертва — `.comparison-ai-block`, тест: [comparison-ai-block-overflow.test.js](tests/unit/ui/comparison-ai-block-overflow.test.js).

### Domain/калькуляция

- **НДС — НЕ риск-коэффициент** (12.U20). `riskFactor.total` БЕЗ `vatMul`. `costFinal = costBase × (applyRisks ? riskTotal : 1) × vatMul`. НДС применяется независимо от master-toggle рисков. Если приходит новая ось накруток — спросить: это РИСК (опциональная надбавка) или ФАКТ (налог/комиссия)?

- **Number-overflow в qty/cost = `cell.error`** (11.2.3). `Number.isFinite(...)` guard в [calculator.js](js/domain/calculator.js); Infinity/NaN не утекает в агрегаты/CSV.

Строгий `value === 0` для money — анти-паттерн: после 6+ накапливающихся умножений (buffer × inflation × seasonal × schedule × contingency × VAT) float-rounding даёт остатки порядка 1e-15. `isZeroMoney(v)` возвращает true для `0/+0/-0/NaN/Infinity` и `|v| < 0.005`. Тест: [money-epsilon.test.js](tests/unit/utils/money-epsilon.test.js).
- **`CURRENT_SCHEMA_VERSION` — re-export от `LATEST_SCHEMA_VERSION`** (12.U32 #5, [constants.js](js/utils/constants.js)). Литерал убран. Bump происходит автоматически при добавлении нового шага миграции в `MIGRATIONS[]`. Single source of truth — `js/state/migrations.js`.
- **`store.getSubscriberCount()` — публичный геттер для leak-detection** (12.U32 #2, [store.js](js/state/store.js)). Integration-тест [store-no-leak.test.js](tests/integration/store-no-leak.test.js) гарантирует, что count не растёт между rerender-циклами. Vanilla не имеет React-cleanup'а — забытый `subscribe()` без `unsubscribe()` приводит к OOM в long-running session.
- **`style:` в `el(...)` запрещён для user-input значений** (12.U32 #3, [style-no-user-input.test.js](tests/unit/architecture/style-no-user-input.test.js)). Линтер (33 файла) запрещает в `style:` любые `Q.<id>` / `answers.*` / `e.target.value` / `draft.*` / `item.name|vendor|description` / `q.title` / `calc.name` / `search`. Защита от CSS-injection при `style-src 'unsafe-inline'` (которое осознанно оставлено для динамических `width:${pct}%` и `background:CATEGORY_COLORS[cat]` — нельзя вынести в классы без сервера/nonce).
- **Все settings-коэффициенты в `riskFactor()` идут через `numWithDefault(value, fallback)`** (12.U31 B.6). Helper в начале [calculator.js](js/domain/calculator.js) обрабатывает `null`/`undefined`/`NaN` единообразно: `null/undefined → DEFAULT`, `Number(x) NaN → DEFAULT`, иначе `Number(x)`. Раньше `Number(bufferTask) || 0` маскировал `NaN→0` и игнорировал `DEFAULT_BUFFER_TASK`, асимметрично с `kInflation/kSeasonal` (через `?? DEFAULT`). Импортированный из ручной правки JSON legacy-расчёт без буферов терял 30% надбавку молча.
- **`validateSettings` reject-ит `null` для критических полей** (12.U31 B.2). `vatRate=null`, `kInflation=null`, `vatEnabled=null` отвергаются явно (`Number(null)=0` → тихое обнуление НДС). `undefined` ОК — `numWithDefault` подставит DEFAULT.
- **`validateCalculation` проверяет per-question type-check answers** (12.U31 B.4). После обработки `dictionaries.questions` строится `qById`-map и каждое значение в `calc.answers` проверяется на ожидаемый тип (`number`/`boolean`/`select`/`multiselect`/`text`). `null` для любого типа = «Не знаю», допустимо. Защита от `[object Object]` в PDF и от `0` в формулах через silent `toNum(Object)=0`.
- **`VALIDATION.ANSWER_STR_MAX = 4096`** (12.U31 B.3). Любое строковое значение в `calc.answers[id]` длиннее 4KB — ошибка валидации. Защита от 10MB-нагрузки в JSON-импорте, которая уронит localStorage пользователя на следующем `commitActiveCalc`.
- **DSL-формулы ограничены `FORMULA_MAX_DEPTH = 64`** (11.2.2). Глубже — `FormulaError`, не RangeError. Если seed-формула превысит — сигнал переписать.
- **`S.standSizeRatio.<STAND>` контекст-зависимый** (12.U12, schema v3). Для item с `dashboardResource` (CPU/GPU/RAM/SSD/HDD/S3) `buildContext` подменяет общий ratio на per-resource. Item без `dashboardResource` (Услуги/Лицензии/Безопасность/Трафик) использует общий. Семантика: `S.standSizeRatio.DEV` в формуле `cpu-vcpu-shared` для DEV даёт CPU-ratio для DEV; в формуле `ram-gb` для DEV — RAM-ratio.
- **«Объёмы ресурсов» на дашборде учитывают `applyRiskFactors`** (12.U7). При `applyRisks=true` qty домножается на `bufferFactor × seasonalMul × scheduleMul × contingencyMul` (БЕЗ vatMul/inflation — это финансовые, не capacity-факторы).

- **Persist UI-state**: F5 не должен сбрасывать пользовательскую вкладку/период/фильтры. Любая view-настройка → localStorage или `calc.view`. Дефолт `null` = «не сохранено» (UI выбирает дефолт).
- **UI-toggle, меняющий ВСЕ суммы расчёта — это параметр расчёта** (`calc.settings`), не view-настройка. View — это период/стенды-фильтр/sub-tab.
- **`toggleStand` через `commit()`** (12.U30 1.3). Изменение `calc.view.disabledStands` обязано идти через `commit()` autosave, иначе F5 сбрасывает фильтр.
- **Все ключи localStorage — через `STORAGE_KEYS`** (11.2.5). Hardcoded литералы запрещены.
- **`resetAll`/`listKeys` в [storage.js](js/services/storage.js) — whitelist через `Object.values(STORAGE_KEYS)`** (12.U31 C.1). Раньше хардкод 6 ключей пропускал 8 ключей UI-state (questionnaireOpenSections, comparisonSort, *CollapsedCats), добавленных в 12.U1/U25/U27/U28/U29 — после «Сбросить всё» они оставались orphan и применялись к новым расчётам. При добавлении нового `STORAGE_KEYS.X` — автоматически охватывается. Линтер: [storage-whitelist.test.js](tests/unit/services/storage-whitelist.test.js).
- **`uuid()` бросает без crypto API** (10.4.4). Math.random fallback удалён.
- **`importCalcFromFile` НЕ делает silent rename** (11.1.4). При коллизии без явного `onDuplicate` возвращает `{ ok:false, reason:'duplicate', ... }` — UI открывает [duplicateImportModal](js/ui/modals/duplicateImportModal.js).
- **`BUNDLE_MAJOR = 1`**. Breaking changes формата — bumping major + обновление `BUNDLE_VERSION`.

### Безопасность

- **`trustedHtml` strict (10.3.1)**. `el(..., { html: ... })` бросает. Используй `el(..., { trustedHtml: trustedHtml(svgString) })` для доверенных строк или `text:` для пользовательского ввода.
- **CSV-инъекция (10.1.1)**. Любая ячейка экспорта, начинающаяся с `=`/`+`/`-`/`@`/`\t`/`\r`, через `csvSafeQuote(value, delimiter)` из [csvExport.js](js/services/csvExport.js).

- **CSP в `<head>` index.html** (11.3.1). Любая правка с inline-скриптами/external CDN — расширяет CSP. Дефолт `'self'`.
- **escapeHtml**: не оставлять backticks (` ` `) — превратятся в Markdown в snackbar.

Между карточками тот же бейдж — это маркер scope, не дубль. Тест: вырвать карточку из контекста — пользователь должен по ней одной понять режим.

- **Section accordion: `<button type="button">`, не `<div>`** (12.U1). Иначе ломается keyboard-фокус и `aria-expanded`.
- **Edge-cases в number-вопросах — обязательно в `description`** (12.U3). Если пользователь спросил «как считается X при Y=0?» — description не справился.

- **`trustedHtml` strict (Этап 10.3.1)**. Любой `el(..., { html: ... })` теперь бросает. Используй `el(..., { trustedHtml: trustedHtml(svgString) })` для доверенных строк (SVG-иконки, `renderMarkdown`, `highlightFormula`) или `el(..., { text: userInput })` для пользовательского ввода. `setTrustedHtml(node, trustedHtml(value))` вместо устаревшего `setHtml`.
- **CSV-инъекция (Этап 10.1.1)**. Любая ячейка экспорта, начинающаяся с `=`/`+`/`-`/`@`/`\t`/`\r`, должна идти через `csvSafeQuote(value, delimiter)` из [js/services/csvExport.js](js/services/csvExport.js) — он добавит `"'<value>"`. В round-trip префикс `'` остаётся в данных (документированный компромисс). Не пиши локальные `quote()` — используй экспорт.
- **`MigrationError` — отдельный класс**, не generic Error (Этап 10.1.3). Контроллеры (`openCalc`/`initFromStorage`/`importCalcFromFile`) и `applyStateBundle` ловят его специально и отдают `{ ok:false, reason:'migration', errors:[{calcId, step, message}] }`. Per-step миграция работает на deep-copy — partial-mutated calc не доходит до store.
- **`commitActiveCalc` — атомарность по 2 ключам** (Этап 10.1.5). Сначала `saveCalc(calc)`, затем `saveCalcList(updatedList)`. На сбое второго — best-effort откат `saveCalcList(backupList)`. Не пиши собственный persist в обход этой обёртки.
- **`getStorage()` probe** ([js/services/storage.js](js/services/storage.js)) делает `setItem('__test__', ...)` перед каждой записью и переключает на in-memory fallback при сбое probe'а. Тесты на quota не должны роняться по count — фильтруй по конкретному ключу или по префиксу `calc.*` (см. `installSpyStorage` в `tests/integration/calc-persistence-atomicity.test.js`).
- **`writeJson` глотает throw** ([js/services/storage.js](js/services/storage.js)). Возвращает `false` при quota — не throws. Поэтому проверяй возвращаемое значение и обрабатывай через `setPersistStatus('error', ...)`.
- **`uuid()` бросает без crypto API** (Этап 10.4.4). Math.random fallback удалён. Если приложение запущено в среде без crypto.getRandomValues — упадёт сразу при создании первого расчёта (намеренно).
- **`BUNDLE_MAJOR = 1`**. При breaking changes формата bundle (новые обязательные поля в схеме calc, изменение DSL) — bumping до 2 + обновить `BUNDLE_VERSION` в `bundleExport.js`. Старые приложения отвергнут новый bundle с понятным сообщением.

Используй `commitActiveCalc`/`commitNewCalc`/`commitCalcRename`/`commitMigratedCalc` из [calcPersistence.js](js/services/calcPersistence.js). Прямая пара `persist.saveCalc + persist.saveCalcList` в controllers — нарушение контракта атомарности, найди обёртку или добавь новую через `_atomicCalcAndListWrite(calc, listBuilder)`.
- **`_notify` ловит throw subscriber'ов** (Этап 11.1.2). Subscriber может выбросить — остальные всё равно вызываются, ошибка летит в `console.error`. Не полагайся на «throw в subscriber'е остановит цепочку» — это больше не так.
- **`debounce` имеет `.flush()` и `.cancel()`** (Этап 11.1.3). beforeunload-listener в boot вызывает `flushPendingCommit()` для autosave-debounce. Если делаешь свой debounce-поток для записи — добавь его в beforeunload-цепочку.
- **`importCalcFromFile` НЕ делает silent rename** (Этап 11.1.4). При коллизии id без явного `onDuplicate: 'replace'|'clone'` возвращает `{ ok: false, reason: 'duplicate', existingId, importedName, preloaded }`. UI должен открывать [duplicateImportModal](js/ui/modals/duplicateImportModal.js) и повторно вызывать с явным выбором.
- **UI НЕ импортирует напрямую `controllers/` или `state/`** (Этап 11.1.5). Все взаимодействия через ctx-методы из [app.js](js/app.js). Layer-linter [tests/unit/architecture/layer-imports.test.js](tests/unit/architecture/layer-imports.test.js) фейлит CI на нарушения. При создании нового UI-файла линтер автоматически подхватит.
- **CSV-импорт цен: `safeUpdates` vs `anomalies`** (Этап 11.2.1). `diff.updates` больше нет. Аномалии (×≥10) — отдельный массив, требуют `confirmAnomalies(anomalies) → Promise<boolean>` callback в `importItemPrices`. Без подтверждения аномальные цены не применяются.
- **DSL-формулы ограничены глубиной 64** (Этап 11.2.2, `FORMULA_MAX_DEPTH`). Глубже — `FormulaError`, не RangeError браузера. Если seed-формула превысит лимит — это сигнал переписать.
- **Number-overflow в qty/cost = `cell.error`** (Этап 11.2.3). Infinity/NaN никогда не доходит до агрегатов, UI или CSV. Проверка `Number.isFinite(...)` в [calculator.js](js/domain/calculator.js).
- **`buildComparisonCsv` живёт в [csvExport.js](js/services/csvExport.js)**, не в app.js (Этап 11.2.4). Все CSV-builders в одном месте.
- **Все ключи localStorage — через `STORAGE_KEYS`** (Этап 11.2.5). Hardcoded литералы запрещены — добавляй константу. `resetAll`/`listKeys` поддерживают весь whitelist.
- **CSP в `<head>` index.html** (Этап 11.3.1). Любая правка, требующая inline-скриптов / external CDN, должна расширять CSP. Безопасный дефолт — `'self'`.
- **Downgrade migration (`schemaVersion > LATEST`) throws MigrationError** (Этап 11.3.2). Раньше calc возвращался as-is — потенциальная порча данных. Теперь явно блокируется.
- **`outline: none` на `:focus*` без замены — запрещено линтером** (Этап 12.1.1, [tests/unit/architecture/a11y-focus.test.js](tests/unit/architecture/a11y-focus.test.js)). При сбросе системного outline обязана быть замена через `box-shadow`/`border`/`outline:<value>` в той же декларации (WCAG 2.4.7). Для `.input` есть split: `:focus` (mouse — только border-color) и `:focus-visible` (keyboard — добавляется ring).
- **Touch-targets ≥44×44 на сенсорных устройствах** (Этап 12.1.2, WCAG 2.5.5). Любая новая интерактивная иконка-кнопка должна попадать в `@media (pointer: coarse), (max-width: 720px)` с `min-height: var(--touch-target)` (или явные `width/height: var(--touch-target)` для квадратных). Линтер: [tests/unit/architecture/touch-targets.test.js](tests/unit/architecture/touch-targets.test.js).
- **Длительные async-операции — через `withLoadingButton(triggerEvent, asyncFn)`** (Этап 12.1.3, [js/app.js](js/app.js)). Любой новый `import*` / `export*` / `print*` ctx-метод обязан принимать `triggerEvent` первым параметром, оборачивать тело в `withLoadingButton`. UI-onClick передаёт event: `(e) => ctx.foo(e)`. ## ✅ Pre-commit чек-лист (быстрая самопроверка)

Перед каждым коммитом пройтись по 12 пунктам. Подробные правила — в разделах выше; здесь — только быстрый сканер. Если хоть один пункт под вопросом — НЕ коммитить, сначала разобраться.

1. **`npm test`** зелёный. Если упали — НЕ маркировать «исправлено», пока не зелёные.
2. **`npm run syntax-check`** зелёный (`node --check` на всех `js/**/*.js`).
3. **Ничего не «забыто» вне коммита**: `git status` чистый по релевантным файлам, screenshots от Playwright НЕ в корне (только в `.playwright-mcp/`), временные `.bak`/`.old`/`*_v2`/JSON-дампы — в `d:/tmp/` или `$TEMP`.
4. **Слои не нарушены**: UI не импортирует напрямую `controllers/` или `state/`; ловит [layer-imports.test.js](tests/unit/architecture/layer-imports.test.js).
5. **Все ключи localStorage** идут через `STORAGE_KEYS` (никаких хардкод-литералов); ловит [storage-whitelist.test.js](tests/unit/services/storage-whitelist.test.js).
6. **Любой `style:` в `el(...)`** — только из доверенного кода (CATEGORY_COLORS, pct), не из `Q.*`/`answers.*`/`item.name`/`search`; ловит [style-no-user-input.test.js](tests/unit/architecture/style-no-user-input.test.js).
7. **Никаких `el(..., { html: ... })`** для произвольных строк — только `trustedHtml(...)` или `text:`. CSP запрещает `'unsafe-eval'`.
8. **Никаких `eval` / `new Function` / `setTimeout(string)` / `.innerHTML = userInput`**.
9. **Magic-числа времени и лимитов** — через `constants.js` (`*_MS`, `*_MAX_BYTES`), не через литералы в коде.
10. **Денежные сравнения** через `isZeroMoney(x)` (EPSILON_KOPECK), не через `x === 0` — после 6+ накопленных умножений float даёт остатки 1e-15.
11. **Settings-коэффициенты** — через `numWithDefault(value, fallback)` в `calculator.js`, не `Number(x) || 0` (маскирует `NaN→0` и игнорирует DEFAULT).

### Что НЕ делать (anti-patterns, повторявшиеся жертвы)

- **`overflow: auto/hidden/scroll` на ancestor таблицы** ломает sticky-thead. Три жертвы уже были (`.app-main`, `.items-table-wrap`, `.comparison-table-wrap`). Новые `*-table-wrap` — БЕЗ `overflow:` вообще.
- **`table-layout: fixed` без `<colgroup>`** на широких таблицах — числа вылезают из ячеек на 1366px ноутбуке. Использовать default auto-layout.
- **`flex-wrap: wrap` + фиксированная `height`** — анти-паттерн (вторая строка наложится). Использовать `min-height`.
- **Grid-item с табличным контентом БЕЗ `min-width: 0`** (13.U11). Дефолтный `min-width: auto` на grid-item означает «расти по min-content ребёнка». С таблицей `white-space: nowrap` это раздувает трек шире своей `1fr`-аллокации, item пролезает в соседнюю карточку. Симптом: «карточка лопается, числа вылезают за рамку». Жертва — `.comparison-ai-block`.
- **Анимация на пересоздаваемом узле** = flicker. Использовать класс-маркер «свежесть» (`.modal-overlay-fresh`).
- **`<select>` + установка `value` ДО добавления `<option>`** — value не применится. Сначала children, потом value.
- **`outline: none` на `:focus*` без замены** — WCAG 2.4.7. Замена через `box-shadow`/`border`/`outline:<value>` обязательна.
- **`display: none` на зависимых полях** master-toggle. Использовать `disabled`+приглушение через `dependsOnUnmet` — пользователь видит, ЧТО появится при включении master.
- **Прямая пара `persist.saveCalc + persist.saveCalcList`** в контроллерах — нарушает атомарность. Только через обёртки `commit*Calc` в [calcPersistence.js](js/services/calcPersistence.js).
- **Хардкод `'1500'`/`'350'`/`'200'` ms** в коде — атавизм этапа 12.U33 и ранее. Все тайминги — в `constants.js` секция «Прочие тайминги UI / IO».
- **Эмодзи в UI** — запрещены, только inline-SVG из [icons.js](js/ui/icons.js) (Lucide). В .md и комментариях кода — допустимы.
- **Tooltip = повторение visible label** — шум + ложное обещание. Tooltip только при ДОПОЛНИТЕЛЬНОЙ инфе.
- **Семантические дубли полей** (Q.id ↔ settings ↔ UI-toggle) — критическая ошибка. Перед добавлением — grep по существующим title/id/семантике.

### Стандарты JS

- **ES-модули, ES2020+**, без сборщика, без npm-runtime-зависимостей.
- **`const`/`let`**, никаких `var` (линтером не покрыто, но в проекте 0 случаев).
- **`for..of`**/`map`/`filter` предпочтительнее индексных циклов. `for..in` — нет.
- **`async`/`await`** для всех Promise-цепочек. Не `.then().catch()` без причины.
- **Шапка файла** — короткий комментарий о назначении. Без JSDoc-простыни.
- **Комментарии — про «почему», не «что»**. Если идентификатор хорошо назван — комментарий не нужен.
- **`Number.isFinite(x)` guard** перед агрегацией финансовых значений (защита от Infinity/NaN).
- **`escapeHtml(value)`** для любого user-content, попадающего в `text:` атрибут или сообщение snackbar (backtick → Markdown bug).

### Стандарты CSS

- **CSS-переменные** в `:root` (`--color-*`, `--space-*`, `--font-*`, `--touch-target`, `--t-fast`). Не дублировать литералы.
- **Темизация через `[data-theme="..."]`**, не через class swap палитры в JS.
- **`@layer`** для управления каскадом (а не порядок импортов).
- **`@container`** для компонентной адаптивности параллельно с `@media` viewport.
- **`:focus-visible`** для keyboard-фокуса; `:focus` оставлять только для базового border-color.
- **Touch-target ≥44×44** на `(pointer: coarse)` (`--touch-target`).
- **`prefers-reduced-motion`** обнуляет animations/transitions.
- **`tabular-nums`** на денежных колонках. **Кириллический контент** — учитывать в ширинах.
- **Нейминг — flat-классы по слоям** (`forms.css`, `dashboard.css`, `comparison.css`). НЕ BEM. НЕ препроцессоры. НЕ Tailwind.
- **`!important`** — только для utility-классов (`.hidden`) и `prefers-reduced-motion`-overrides.

### Стандарты HTML

- **`<script type="module">`** только. Никаких inline-скриптов кроме CSP-разрешённых.
- **Семантические теги**: `<main>`, `<nav>`, `<aside>`, `<header>`, `<footer>` — где есть смысл. Layout создаётся динамически в [index.js](js/ui/index.js); статика в [index.html](index.html) — только boot-fallback.
- **`aria-labelledby`** на модалках. **`role="status" aria-live="polite"`** на snackbar-стеке.
- **`id="main-content" tabindex="-1"`** — обязательное требование skip-link (живёт в JS-рендере [index.js:78-85](js/ui/index.js#L78-L85)).
- **CSP** в `<meta>` — никаких `'unsafe-eval'`, `'unsafe-inline'` в `style-src` оставлен осознанно (динамические `width:${pct}%` + `background:CATEGORY_COLORS`), документировано в [index.html:5-23](index.html#L5).
